# Endless

> **AI-автопилот для личных финансов поверх [ZenMoney](https://zenmoney.ru/).**
> Автоматически категоризирует каждую транзакцию, держит envelope-бюджет
> в порядке и не даёт забросить денежный дневник на полпути.

**Живое демо:** <https://endless.vyroslo.ru> — пять вымышленных россиян
и 5-летняя синтетическая история, сгенерированная локально в твоём
браузере. Ничего не уходит на сервер.

---

## Зачем это

ZenMoney — отличный импортёр транзакций: банки присылают, приложение
собирает. Но «думающая» часть — всё ещё ручная: каждую операцию нужно
категоризировать, каждый месяц — сводить бюджет, каждую крупную трату —
помнить в контексте. 90% людей бросают ZenMoney/YNAB за месяц, потому
что трение копится быстрее, чем инсайты.

Endless забирает себе скучную половину:

- **Автокатегоризация.** Мерчант, MCC, паттерны получателей и
  LLM-fallback раскидывают транзакции по категориям. Вместо ежедневной
  рутины ввода — короткий digest.
- **Envelope-бюджет.** В стиле YNAB: «каждый рубль назначен до траты».
  Мультивалюта и true-expense конверты (авто, ремонт, годовая
  страховка).
- **Детект переводов.** Между своими счетами, P2P родным, round-trip
  возвратов — не попадают в «траты».
- **Горизонт Forever.** 3/5/10-летний прогноз капитала в трёх
  сценариях (консервативный / реалистичный / оптимистичный) из
  фактической ставки сбережений, а не из «мечты».

Тебе остаётся **одно** — подтверждать неоднозначные транзакции один
раз, чтобы система училась. Через пару недель ежедневный обзор
занимает меньше минуты.

## Структура репозитория

Три независимых пакета, каждый со своим `package.json` и
`node_modules`; корневого workspace нет:

```
EndlessZenMoney/
├── app/             # Node CLI — ежедневный sync + категоризация + digest
├── endless-web/     # React веб-приложение — конверты, аналитика, демо-режим
├── zenmoney-api/    # Общий клиент ZenMoney API (vendored fork)
└── research/        # PDF'ки и заметки, сформировавшие продуктовое видение
```

### `app/` — CLI

CLI на Commander, который запускается из крона или горячей клавишей.

```
$ endless sync
  Подключение к ZenMoney... OK
  Загрузка данных... OK (47 транзакций)
  Авто-категоризировано: 41
  Нужна помощь: 6
  [Группа 1/4] 3 похожих транзакций:
    04-22  Пятёрочка  −1 247 ₽
    04-21  Пятёрочка  −892 ₽
    04-20  Пятёрочка  −1 540 ₽
    Предлагаю: Продукты (96%) — same-merchant frequency match
    [Enter] Принять  [c] Изменить  [s] Пропустить
  ...
  📊 Digest:
  Income:   +80 000 ₽
  Spent:    −47 120 ₽
  Savings:   18%
```

Кластеризация похожих непонятных транзакций и подтверждение пачками —
единственное, что превращает «ежедневные 60 секунд» в устойчивую
привычку.

### `endless-web/` — веб-приложение

Envelope-бюджет как PWA на React + Redux Toolkit + MUI с расширениями:

- Демо-режим, который загружает сгенерированную 5-летнюю историю
  персонажа без сетевых вызовов.
- PIN-хранилище токена: single-user деплой без OAuth-редиректа на
  каждый сеанс.
- Страница прогноза капитала (3/5/10 лет, три сценария).
- Дополнительная аналитика: savings rate, true-expense конверты,
  раскладка капитала по типам счетов.

Веб-пакет — форк [Zerro](https://github.com/ardov/zerro) под GPL-2.0;
полные upstream copyright-уведомления сохранены в [LICENSE](LICENSE).

### `zenmoney-api/`

Минимальный TypeScript-клиент для diff-sync API ZenMoney. `app/` и
`endless-web/` зависят от него через workspace-link.

## Быстрый старт

### Требования

1. **Node 20+** и **pnpm 9+**
2. **Аккаунт ZenMoney** с импортированными транзакциями
3. **OAuth-приложение ZenMoney** — бесплатно, self-service на
   <https://api.zenmoney.ru/>. Скопируй `consumer_key` и
   `consumer_secret`.
4. *(Только для CLI, опционально)* **Ключ Anthropic API** для
   LLM-категоризатора. Без него CLI откатывается в rules-only режим.

### Установка

```bash
git clone https://github.com/d3xr/endless.git
cd endless
# Каждый пакет ставится независимо — корневого workspace нет.
( cd zenmoney-api && pnpm install && pnpm build )
( cd app          && pnpm install )
( cd endless-web  && pnpm install )
```

### Запуск CLI

```bash
cp app/.env.example app/.env
# Отредактируй app/.env — впиши ZM_TOKEN и ANTHROPIC_API_KEY
cd app
pnpm sync                # первый sync — вся история + интерактивный обзор
pnpm sync                # следующие синки — инкрементальные
pnpm status              # показывает время последнего sync
```

> **Как получить токен ZenMoney для CLI:** проще всего — запустить
> веб-приложение локально, войти через OAuth и скопировать токен из
> `localStorage.zm_access_token` в DevTools. Для постоянного токена
> — OAuth-flow напрямую против <https://api.zenmoney.ru/>.

### Запуск веб-приложения локально

```bash
cd endless-web
cp .env.development .env.development.local
# Отредактируй .env.development.local — впиши REACT_APP_CLIENT_ID/SECRET
pnpm dev
# http://localhost:3000 — кнопка «Войти», OAuth возвращает на localhost
```

Все режимы продакшен-сборки (OAuth / PIN-locked / публичное демо)
описаны в [endless-web/DEPLOY.md](endless-web/DEPLOY.md).

## Демо-режим и персоны

`endless.vyroslo.ru` — это публичная сборка с
`REACT_APP_DEMO_ONLY=true`: в бандл не вкомпилирован ни один auth-путь.
Посетитель выбирает одного из пяти вымышленных россиян, и браузер
локально генерирует для него 5-летнюю историю через Mulberry32 PRNG.
Всё лежит в IndexedDB и стирается при смене персонажа или очистке
вкладки.

Пять персон покрывают спектр по доходу намеренно:

| Персона             | Уровень        | Роль                         | Почему интересно                            |
|---------------------|----------------|------------------------------|---------------------------------------------|
| Марина Петрова      | Низкий         | Кассир, Саратов              | Cash-доминирование, живёт от зарплаты       |
| Алексей Смирнов     | Ниже среднего  | Бухгалтер, Воронеж           | Классический офисный семьянин с ипотекой    |
| Ольга Кузнецова     | Средний        | Офис-менеджер, Екатеринбург  | Одинокая, дисциплинированный сберегатель    |
| Дмитрий Волков      | Выше среднего  | Senior Backend, Москва       | IT-зарплата, двое детей, маркетплейсы       |
| Николай Рябинин     | Высокий        | Владелец автосервиса, Краснодар | Малый бизнес, cash flow, дача и баня     |

У каждой персоны — подробная биография (хобби, характер, боли,
бэкстори) прямо внутри пикера, плюс события по годам (повышения,
отпуска, крупные покупки), которые генератор превращает в реальные
транзакции.

См. [endless-web/src/demoData/personas/](endless-web/src/demoData/personas/) —
`profiles.ts` содержит биографии, `generator.ts` — движок транзакций.

## Безопасность

В этом репозитории ничего не хранит и не передаёт реальные данные
пользователей. Все три режима сборки держат токен локально:

- **Dev:** `REACT_APP_DEV_TOKEN` в `.env.development.local` (только
  localhost, никогда не попадает в продакшен-бандл).
- **Prod с OAuth:** каждый посетитель логинится своим ZenMoney. Токен
  лежит в `sessionStorage` (стирается при закрытии вкладки).
- **Prod с зашифрованным блобом:** AES-GCM ciphertext в JS-бандле,
  PBKDF2 (200k итераций) из PIN. Неправильный PIN фейлится
  криптографически, а не «выглядит по-другому». Rate-limiter на
  клиенте добавляет трение (5 попыток → 60s cooldown).
- **Demo:** никаких секретов вообще, всё работает в браузере.

Модель зашифрованного блоба — это лежачий полицейский, а не
нормальная защита: 4-значный PIN при PBKDF2 на 200k итераций
брутфорсится примерно за 1.5 часа. Хочешь настоящую защиту — ставь
6+ знаков, или прикрывай деплой nginx Basic Auth / VPN / Tailscale.

### Изоляция демо и приватного инстансов

Публичный демо (`endless.vyroslo.ru`) и приватный инстанс
(`endless-private.vyroslo.ru`) живут в **отдельных docker-контейнерах**
на одной VPS:

- У каждого контейнера read-only bind-mount только своей html-папки
  (`/srv/endless/demo/html` и `/srv/endless/private/html`); папки друг
  друга физически не видны в mount namespace.
- Отдельные bridge-сети (`demo_net`, `private_net`).
- `cap_drop: ALL` + минимальный re-add для nginx, `no-new-privileges`,
  `read_only: true`, tmpfs на writable путях.
- Хостовой nginx — чистый TLS-terminating reverse proxy на
  `127.0.0.1:8081` и `127.0.0.1:8082`.

Даже если кто-то найдёт RCE в демо-контейнере, приватный бандл
физически вне его mount namespace. Плюс в демо-бандле токенов нет
вообще, а приватный токен лежит только в localStorage браузера
пользователя — на сервере его нет.

## Контрибьют

Endless — в основном персональный проект, выложенный в публичный
доступ. Приветствуется:

- Issues с описанием поведения, которое хотелось бы увидеть по-другому.
- PR'ы с багфиксами, новыми персонами, языковыми портами.
- Fork-and-run — CLI и веб-приложение самостоятельны, бери что нужно.

Для крупных изменений — сначала issue, чтобы обсудить направление.
Переводы UI и персон особенно приветствуются, если хочешь
прогнать приложение по не-RU банковским данным.
